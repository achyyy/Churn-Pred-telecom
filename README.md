# Churn-Pred-telecom
# Customer Churn Prediction

A machine learning project that predicts whether a telecom customer is likely to churn.  
Built using pandas, scikit-learn, XGBoost, and SHAP for model explainability.

## Project Overview

This notebook explores the Telco Customer Churn dataset, performs preprocessing and feature encoding, trains classification models, and uses SHAP to explain the final model predictions.

## Dataset

- Source: Telco Customer Churn dataset
- Expected file location: `data/WA_Fn-UseC_-Telco-Customer-Churn.csv`

If the dataset is not included in the repository, download it and place it in the `data/` folder before running the notebook.

## Workflow

1. Load and clean the dataset
2. Perform exploratory data analysis
3. Encode categorical features
4. Handle class imbalance using SMOTE
5. Train models and tune Random Forest
6. Evaluate performance using accuracy, recall, confusion matrix, and ROC-AUC
7. Explain predictions using SHAP

## Key Findings

- New customers are more likely to churn
- Higher monthly charges are associated with higher churn
- Month-to-month contracts show higher churn risk
- Electronic check users tend to churn more often

## SHAP Explainability

This project includes SHAP visualizations to make the model easier to interpret:

- Bar chart: shows the most important features overall
- Beeswarm plot: shows how each feature affects churn predictions

These charts help explain why the model predicts churn for a customer.
